Accessibility

Last updated: March 2026 — We are committed to making Forum Creation usable by everyone.

Our Commitment

Forum Creation aims to be accessible to all users, including those who rely on assistive technologies such as screen readers, keyboard navigation, or high-contrast display modes. We strive to follow the Web Content Accessibility Guidelines (WCAG) 2.1 as a baseline for our design decisions.

Keyboard Navigation

All core functionality on this site can be accessed using a keyboard alone:

  • Use Tab to move forward through interactive elements
  • Use Shift + Tab to move backward
  • Use Enter or Space to activate buttons and links
  • Use Escape to close modals and clear the search input
  • Use Arrow keys to navigate radio button groups in forms

Screen Reader Support

We use semantic HTML elements (nav, main, aside, h1–h3, button, etc.) so that screen readers can correctly interpret the structure of our pages. Interactive controls include descriptive labels and ARIA attributes where needed.

Color and Contrast

Our dark theme is designed with contrast ratios in mind. Primary text against the page background meets WCAG AA contrast requirements. Accent colors are used to highlight interactive elements and active states, not to convey information exclusively through color.

Text and Font Size

All text on the site is rendered with relative units and respects your browser's font size settings. You can increase or decrease text size using your browser's built-in zoom or text size controls without breaking the layout.

Images and Media

Images uploaded by users are presented with appropriate structural context. We encourage users to include descriptive context in their posts when sharing images so that all community members can engage with the content.

Mobile and Touch Accessibility

The site is fully responsive and designed for touch interaction. Touch targets are sized to be comfortably tappable. The layout adapts to different screen sizes and orientations.

Known Limitations

We are continuously working to improve accessibility. Current areas we are actively improving include:

  • More comprehensive ARIA live region announcements for dynamic content updates
  • Skip-to-content links for keyboard users
  • Enhanced focus indicators in all themes

Feedback

If you encounter an accessibility barrier or have suggestions for improvement, we want to hear from you. Please use the report button on any post to reach our moderation team, or describe the issue in a post in the General forum. We take accessibility feedback seriously and will work to address issues promptly.
